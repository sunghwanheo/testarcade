# Player Select — 인원 선택 기능 설계

**날짜:** 2026-04-15  
**상태:** 승인됨

---

## 배경 및 목적

아이들이 게임 카메라 앞에 통제 없이 들어오는 문제를 해결하기 위해, 각 게임 시작 전 몇 명이 플레이할지 명시적으로 선택하는 기능을 추가한다. 인원 선택 후 게임은 해당 인원수를 `window.playerCount`에 저장하고, 이후 트래킹 로직이 이 값을 참조할 수 있도록 준비한다.

---

## 범위

### 대상 게임 및 최대 인원

| 게임 | 폴더 | 최대 인원 |
|------|------|----------|
| 풍선팡팡 | `balloon_pang/` | 3인 |
| 뿡뿡이 | `pung/` | 3인 |
| 얼음깨기 | `ice_breaker/` | 3인 |
| 두더지 잡기 | `mole/` | 2인 |
| 와구와구 | `lion/` | 변경 없음 (1인 고정) |

### 이번 작업 범위

- 인원 선택 UI 표시 및 선택값 저장 (`window.playerCount`)
- 각 게임 트래킹 로직과의 연결은 **별도 작업**으로 진행

---

## 아키텍처

### 신규 파일

**`/player-select.js`** (루트)

- 각 게임 HTML의 `<head>`에 `<script src="../player-select.js">` 추가
- 페이지 로드 시 `data-max` 속성을 읽어 토글 버튼 자동 렌더링
- `window.playerCount` 에 선택값 저장 (기본값: `1`)
- 한/영 언어 지원 (`getLang()` 활용)
- 별도 JS 호출 불필요 — HTML 삽입만으로 동작

### 수정 파일

각 게임의 기존 시작 화면 HTML에 아래 한 줄 삽입 (기존 시작 버튼 위):

```html
<!-- data-max: 해당 게임의 최대 인원수 -->
<div id="player-select" data-max="3"></div>
```

`player-select.js`가 `data-max` 값을 읽어 버튼 개수를 결정하고 자동 렌더링한다. JS 파일 수정 불필요.

## UI 설계

### 인원 선택 토글

```
👥 인원 선택
┌────┐ ┌────┐ ┌────┐
│ 1인 │ │ 2인 │ │ 3인 │   ← max:2이면 버튼 2개
└────┘ └────┘ └────┘
(기본: 1인 활성화 상태)
```

- 선택된 버튼만 하이라이트 (활성화 스타일)
- 기존 "게임 시작" 버튼은 그대로 유지
- 시작 버튼 클릭 시 `window.playerCount` 값을 읽어 게임 시작
- 터치 친화적 크기 (아이들 대상)
- 스타일은 각 게임 기존 UI 톤에 맞게 통일

### 언어별 텍스트

| | 한국어 | 영어 |
|--|--------|------|
| 레이블 | 인원 선택 | Players |
| 버튼 | 1인 / 2인 / 3인 | 1P / 2P / 3P |

## 데이터 흐름

```
게임 페이지 로드
  → player-select.js 초기화
  → #player-select div에 토글 버튼 렌더링
  → window.playerCount = 1 (기본값 설정)
  → 사용자가 인원 선택
  → window.playerCount 업데이트
  → 기존 시작 버튼 클릭
  → 게임 시작 (playerCount 값 사용 가능)
```

## 제외 사항

- 트래킹 로직 수정 (게임별 별도 작업)
- 와구와구(`lion/`) 수정 없음
- 인원별 개인 점수 분리 없음 (점수는 공유)
- 로비(index.html) 수정 없음
